*link*Progress Indicator

The **Progress Indicator** Widget can be used to inform users about the status of background processes such as saving updates or fetching data from an external source by providing an animated overlay on its parent element.

*link*Basic

The parent element's height must be defined. For example: `100%` or `100px`.

By default, when the `innerOverlay` size reaches 90% of the parent's height and the `size` property is `default`, its size will be changed to `medium` or `small`. You can change the threshold by setting `dynamicHeightThreshold` property to a different number to set it to `false` to disable.

*link*Customization

You can customize a Progress Indicator by using below configurations:

- `label`: add a custom label.
- `type`: display the label and circle in vertical or horizontal alignment
- `useLoadingDots`: specify whether display the loading dots after the label.
- `color`: customize the color of the loading indicator.
- `size`: customize the size of the loading indicator (big, medium, small), the default value is big.
- `singleOverlay`: display the single overlay.

*link*Fast Appear

You can set the `fastAppear` property to true to make the Progress Indicator appears faster than normal.

The inner-overlay, the circle and the label will be appear immediately when the Progress Indicator is shown.

By default, there is no delaying. You can delay the appearance of Progress Indicator by setting the delay time to the `openingDelay` property.

*link*Global

Set `global` to true to make the Progress Indicator cover the entire window.

If you're on desktop, press `Esc` to close the **Progress Indicator** after opening it. Otherwise, if you're on a mobile device it will close automatically after 3 seconds.

*link*Accessibility

- The progress indicator will be focused and automatically scrolled into the visible view if the `focusOnOpen` or `global` property is set to `true`.  
  On the contrary, enabling the `scrollIntoView` property will make the indicator scrolling without the need for focus. It's handy if you don't want to be interrupted when the progress indicator appears.
- The `label` should always be set. If there is no label, the **Progress Indicator** will have a hidden text to fully support accessibility. For more detail, please go to [Accessibility](../Basics.Accessibility/index.md).

ProgressIndicatorProps

Property

Type

Description

`className`

`string`

Additional css class names.

`color`

`string`

Custom color for the progress indicator

`dynamicHeightThreshold`

`number | "false"`

Specifies when the widget should adjust its size based on the parent's size.
Set to false to disable.

**@default** 0.9

`fastAppear`

`boolean`

The circle and label appear immediately.

`focusOnOpen`

`boolean`

If it is set to `true` , the Progress Indicator's inner overlay will be focused and scrolled into view automatically.
On the other hand, use `scrollIntoView` to execute scrolling manually.

`global`

`boolean`

If this property is set to `true` :

- The progress indicator will take up the entire screen and won't be tied to any outer container.
- The inner overlay will be automatically focused.

`hideLoadingCircle`

`boolean`

Hide the loading circle

`id`

`string`

Specifies an id that is used for html id attribute.

See [MDN*open\_in\_new*](https://developer.mozilla.org/en-US/docs/Web/HTML/Global_attributes/id "Leave Page")

`innerOverlayVariant`

`OverlayVariant`

Variant for Inner Overlay.

`label`

`ReactNode`

Set this to display the label below the progress indicator

`noTabIndex`

`boolean`

Specifies whether the progress indicator has no tabIndex.
For avoiding HTML Validator:
noTabIndex should be used in case Progress Indicator is called as a descendant of `button` or `a` element

`openingDelay`

`number`

Delay time when opening the ProgressIndicator.

`outerOverlayVariant`

`OverlayVariant`

Variant for Outer Overlay.

`scrollIntoView`

`boolean`

To scroll the current progress indicator into the browser window's visible area without focusing on it.

**Note:** Only works if `focusOnOpen` is not set.

`singleOverlay`

`boolean`

Display single overlay

`size`

`ProgressIndicatorSize`

Size of the Inner Overlay

**@default** "big"

`style`

`CSSProperties`

Additional styles.

`type`

`"horizontal" | "vertical"`

Set this to display the label and circle in vertical or horizontal alignment

**@default** "vertical"

`useLoadingDots`

`boolean`

Specifies whether using the loading dots after the label or not.
**Note:** Loading dots just appears if there is a label.

`wrapperRef`

`RefCallback<HTMLElement>`

The reference of the element wrapping the main content if one exists.
